## Algorithm used by this Figure 7

## 1) State labels for k=4
- Uses `labels_k4_sfcSorted.npy`: one pooled label sequence over all concatenated timepoints.
- States are already relabeled by ascending SFC against global mean SC.

## 2) State SFC values (x-axis values in panel 5)
- Uses `sfc_k4_sorted_GLOBALmean.json`.
- This is one SFC value per state computed vs `SC_global_mean.npy`.

## 3) Subject occupancy distributions (panels 1-4)
- Splits pooled label sequence by `subject_splits.json`.
- For each subject: occupancy per state = state-count / subject-count.
- Groups subjects into `CNT/MCS/UWS` via `condition_splits_grouped_CNT_MCS_UWS.json`.
- Plots violins + points by condition for each state.
- Welch t-tests between conditions with BH-FDR correction.

## 4) SFC vs occupancy panel (panel 5)
- For each condition and state:
  - x = state SFC from `sfc_k4_sorted_GLOBALmean.json` (same for all subjects)
  - y = subject occupancy for that state
- Scatter points are thus repeated in x by state, spread in y by subject occupancy.
- Linear fits are done per condition on these points.

## Key properties of this figure’s algorithm
- **Pooled state fitting** (not subject-local fitting).
- **Global SC reference for state ordering and SFC** (not subject SC).
- Occupancy is subject-specific, but x-axis coupling is state-global.

## Why trends can differ (conscious-condition SC-coupling)

1. **SC reference difference**
- Figure 7: x-axis SFC from global mean SC (`GLOBALmean`).
- Primary TVBToolkit: subject-specific SC for each subject/state.
- This alone can change trend direction/magnitude.

2. **State-definition difference**
- Figure 7: pooled states from all subjects together.
- Primary TVBToolkit: states fitted separately per subject.
- Pooled states can suppress subject-specific structure and create different condition contrasts.

3. **Condition set difference**
- Figure 7 compares `CNT/MCS/UWS` only.
- TVBToolkit run includes `Control/EMCS/MCS/UWS` and sedation strata (after exclusions).

4. **Preprocessing/filtering differences across pipelines**
- Legacy scripts may use variant choices around filtering and Hilbert preprocessing depending on branch/script.
- TVBToolkit audited path uses a consistent Brain-Act legacy-compatible preprocessing config.

5. **ROI-order parity handling**
- TVBToolkit audited run explicitly tested and applied AAL90 FC-side reorder (`aal90_fc`), substantially increasing FC-SC parity.
- If older pipelines did not enforce identical ROI-order assumptions, coupling trends can shift.
